kubernetes标签介绍

用来组织kubernetes的资料，标签可以附加到任何资源的任意键值对，用以选择具有该标签的资源(标签选择器)。标签的key在资源中是唯一的，一个资源可以拥有多个标签。

创建pod定义标签，案例如下

kubia-manual-with-labels.yaml
apiVersion: v1 
kind: Pod 
metadata:
  name: kubia-manual-v2
  labels:
    creation_method: manual
    env: prod
spec:
  containers:
  - image: tomcat
    imagePullPolicy: Never
    name: kubia
    ports: 
    - containerPort: 8080 
      protocol: TCP

查看pod标签

kubectl get po --show-labels
NAME              READY   STATUS    RESTARTS   AGE    LABELS
kubia-manual      1/1     Running   0          2m4s   <none>
kubia-manual-v2   1/1     Running   0          2m4s   creation_method=manual,env=prod

展示有标签的列

 kubectl get po -L creation_method,env
NAME              READY   STATUS    RESTARTS   AGE   CREATION_METHOD   ENV
kubia-manual      1/1     Running   0          4m
kubia-manual-v2   1/1     Running   0          4m    manual            prod

添加标签到pod

kubectl label po kubia-manual  creation_method=manual
修改后如下
kubectl get po -L creation_method,env
NAME              READY   STATUS    RESTARTS   AGE   CREATION_METHOD   ENV
kubia-manual      1/1     Running   0          12m   manual
kubia-manual-v2   1/1     Running   0          12m   manual            prod

修改pod值

kubectl label po kubia-manual-v2 env=debug --overwrite 
修改后如下
kubectl get po -L creation_method,env
NAME              READY   STATUS    RESTARTS   AGE   CREATION_METHOD   ENV
kubia-manual      1/1     Running   0          15m   manual
kubia-manual-v2   1/1     Running   0          15m   manual            debug

仅当resource-version=1时才更新 名为foo的Pod上的label

kubectl label pods foo status=unhealthy --resource-version=1

给 namespace 中的所有 pod 添加 label

kubectl label pods --all status=unhealthy

删除名为“bar”的label

kubectl label pods foo bar-

使用标签选择器列出pod子集

标签选择器允许我们选择标记特定标签的pod子集，并对这些pod进行操作
标签选择器根据资源的以下条件选择资源：
1、包含(或不包含)使用特定键的标签
2、包含具有特定建和值的标签
3、包含具有特定键的标签，但其值与我们指定的不同

使用标签选择器列出pod

列出标签为 creation_method=manual的
kubectl get po -l creation_method=manual
NAME              READY   STATUS    RESTARTS   AGE
kubia-manual      1/1     Running   0          28m
kubia-manual-v2   1/1     Running   0          28m

列出标签为env，无论值是什么
kubectl get po -l env
NAME              READY   STATUS    RESTARTS   AGE
kubia-manual-v2   1/1     Running   0          30m

列出没有env标签的
kubectl get po -l '!env'
NAME           READY   STATUS    RESTARTS   AGE
kubia-manual   1/1     Running   0          31m

注意要用!env必须是单引号。
匹配有如下：
creation_method !=manual:标签为creation_method，值不等于manual
env in(prod,devel),选择env标签，并且值为prod或者devel
env notin(bug,ord)',选择env标签，但其值不是prod或者devel

使用标签选择器删除pod

kubectl delete po -l creation_method=manual

使用标签分配工作节点

kubectl label node foo gpu=true

apiVersion: v1 
kind: Pod 
metadata:
  name: kubia-manual
spec:
  nodeSelector:
    gpu: "true"  指定pod在那个节点上工作
  containers:
  - image: tomcat
    imagePullPolicy: Never
    name: kubia
    ports: 
    - containerPort: 8080 
      protocol: TCP

别的资源的标签操作跟pod标签操作一致，这里不再做重复操作

注解pod

添加和修改pod

kubectl annotate pod kubia-manual-v2  mycompany.com/someannotation="foo far"
kubectl describe  pod kubia-manual-v2
Annotations:  mycompany.com/someannotation: foo far
